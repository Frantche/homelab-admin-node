Use the quickstart when you already know Proxmox, Ansible, SOPS, and Git. Use the full deployment guide when you want the complete path from an empty Proxmox host to a running admin node.

The documentation is organized around the lifecycle of the node:

- **Get started** explains the project, architecture, requirements, and quickstart.
- **Deployment guide** walks through Proxmox, cloud-init, secret zero, the private config repository, lifecycle modes, and first validation.
- **Configuration** documents each major configuration area and service.
- **Operations** covers the CLI, convergence, backup, restore, validation, CI, and troubleshooting.
- **Contributing** explains local development and contribution expectations.
